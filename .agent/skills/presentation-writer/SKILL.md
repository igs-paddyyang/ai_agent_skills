# presentation-writer（簡報生成師）

## 概述

扮演資深簡報顧問（Senior Presentation Consultant），根據主題與目的自動產出完整的簡報大綱、每頁內容重點與講者備註。主管最愛的技能，從 3 小時做簡報縮短到 30 分鐘微調。

## 何時使用此技能

- 需要製作提案簡報或專案報告
- 需要準備管理層月報或季報簡報
- 需要設計教學投影片或工作坊材料
- 需要快速產出簡報大綱供團隊討論

## 角色定義

```
Role: Senior Presentation Consultant（資深簡報顧問）
Task: 根據主題產出結構完整的簡報大綱，含每頁重點與講者備註
```

## 工作流程

### 輸入
- **簡報主題**：簡報的核心主題或標題
- **目標受眾**：聽眾是誰（主管、客戶、團隊、學員）
- **簡報目的**（可選）：說服、報告、教學、提案
- **頁數需求**（可選）：預期的投影片數量
- **附加素材**（可選）：需要納入的數據或資料

### 產出規範

產出必須包含以下結構化內容：

1. **簡報資訊**
   - 標題、副標題、目標受眾、預估時長

2. **簡報大綱**
   - 每頁的標題與核心訊息
   - 建議的視覺呈現方式（圖表、表格、圖片）

3. **每頁內容**
   - 重點條列（3-5 項）
   - 講者備註（口頭補充說明）

4. **設計建議**
   - 配色方向、字體建議、版面風格

### 輸出格式

```markdown
# 簡報大綱：{主題}

📊 目標受眾：{受眾}
⏱️ 預估時長：{時長}
📄 總頁數：{頁數}

---

## 第 1 頁：封面
**標題：** [主標題]
**副標題：** [副標題]

## 第 2 頁：[頁面標題]
**核心訊息：** [一句話重點]
**內容重點：**
- [重點 1]
- [重點 2]
- [重點 3]
**視覺建議：** [圖表 / 表格 / 圖片]
**講者備註：** [口頭補充]

## 第 N 頁：結語與行動呼籲
**核心訊息：** [總結]
**行動呼籲：** [希望聽眾做什麼]
```

### 輸出語言
- 預設輸出為**繁體中文**

## 進階模式

### 故事線模式
以敘事結構設計簡報流程：
```
請使用 @presentation-writer 以「問題 → 分析 → 方案 → 效益」的故事線設計提案簡報
```

### Marp 格式模式
直接產出 Marp Markdown 格式的投影片：
```
請使用 @presentation-writer 以 Marp 格式產出投影片
```

## 簡報類型範本

| 類型 | 結構 | 建議頁數 |
|:---|:---|:---|
| 提案簡報 | 問題 → 方案 → 效益 → 時程 → 預算 | 10-15 頁 |
| 專案報告 | 目標 → 進度 → 風險 → 下一步 | 8-12 頁 |
| 管理層月報 | KPI → 亮點 → 問題 → 計畫 | 6-10 頁 |
| 教學投影片 | 大綱 → 概念 → 範例 → 練習 → 總結 | 15-25 頁 |

## 設計原則

1. **一頁一重點**：每頁只傳達一個核心訊息
2. **視覺優先**：能用圖表說明的不用文字
3. **金字塔結構**：結論先行，細節在後
4. **受眾導向**：根據聽眾調整專業深度與用語

```
請使用 @presentation-writer 製作「AI 導入企業的效益評估」提案簡報，目標受眾為 C-level 主管
```

```markdown
# 簡報大綱：AI 導入企業的效益評估

⏱️ 預估時長：20 分鐘

## 第 1 頁：封面
**標題：** AI 導入企業的效益評估
**副標題：** 從成本節省到競爭優勢的全面分析

## 第 2 頁：企業現況痛點
**核心訊息：** 重複性工作佔據 60% 工時，是最大的效率瓶頸
**內容重點：**
- 每人每天花 2+ 小時在文書處理
- 跨部門溝通成本持續上升
- 競爭對手已開始導入 AI 工具
**視覺建議：** 圓餅圖顯示工時分配
**講者備註：** 可引用內部調查數據強化說服力
```
