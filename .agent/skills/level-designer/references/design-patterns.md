# 關卡設計模式參考

## 環境設計模式

### 1. 線性關卡（Linear）
玩家沿固定路線前進，適合敘事驅動的遊戲。
- 優點：節奏可控、敘事清晰
- 缺點：重玩性低

### 2. 開放式關卡（Open）
玩家自由探索，適合沙盒類遊戲。
- 優點：探索感強、重玩性高
- 缺點：節奏難控制

### 3. 分支式關卡（Branching）
多條路線通往同一目標，適合策略類遊戲。
- 優點：選擇感、策略深度
- 缺點：開發成本高

## 環境危險設計原則

### 可預測性
- 危險要素應有視覺或聽覺預警
- 玩家應能透過觀察學會規避

### 互動性
- 危險要素應能被 Boss 利用或強化
- 玩家也應能反向利用環境對抗 Boss

### 節奏感
- 危險區域與安全區域交替出現
- 避免連續高壓導致玩家疲勞
