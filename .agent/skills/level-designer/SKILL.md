# level-designer（關卡設計師）

## 概述

扮演資深關卡設計師（Senior Level Designer），根據指定主題產出詳細的環境與挑戰設計文件。此技能是 GDD 自動生成鏈的第一環節，其輸出將作為 `@character-creator` 的上下文輸入。

## 何時使用此技能

- 需要為遊戲主題設計關卡環境與挑戰
- 需要產出 GDD 中的「關卡與環境設計」章節
- 作為鏈式 GDD 生成工作流程的第一步驟
- 需要定義環境氛圍、危險要素與玩家目標

## 角色定義

```
Role: Senior Level Designer（資深關卡設計師）
Task: 為遊戲關卡建立詳細的環境與挑戰概述
```

## 工作流程

### 輸入
- **主題（Theme）**：遊戲關卡的核心主題（例如：深海科研站、火山要塞、太空殖民地）

### 產出規範

產出必須包含以下結構化內容：

1. **核心環境氛圍**
   - 場景的視覺風格與整體氛圍描述
   - 光線、色調、音效環境的設計方向

2. **三大環境危險要素**
   - 列出 3 個與主題高度相關的環境挑戰
   - 每個危險要素需描述觸發條件與對玩家的影響
   - 這些危險要素將作為 `@character-creator` 設計 Boss 技能的依據

3. **玩家主要目標**
   - 此關卡中玩家需要完成的核心任務
   - 目標與環境危險的互動關係

### 輸出格式

```markdown
# 關卡設計：{主題}

## 環境氛圍
[場景描述]

## 環境危險要素
### 危險 1：[名稱]
[描述與觸發條件]

### 危險 2：[名稱]
[描述與觸發條件]

### 危險 3：[名稱]
[描述與觸發條件]

## 玩家目標
[核心任務描述]
```

### 輸出語言
- 預設輸出為**繁體中文**

## 鏈式調用說明

此技能的輸出將作為上下文傳遞給下一個技能：

```
@level-designer（本技能）
    ↓ 輸出：環境設計 + 危險要素
@character-creator
    ↓ 輸入：接收環境設計作為背景
    ↓ 輸出：與環境互動的 Boss 設計
GDD 組裝
    ↓ 合併兩者產出為完整 GDD
```

## 設計原則

參考 `game-design` 技能的核心原則：

1. **主題一致性**：所有環境元素必須與主題高度相關
2. **危險互動性**：環境危險要素應能與 Boss 技能產生互動（供 character-creator 參考）
3. **玩家心流**：難度設計遵循「太難→挫折、太簡單→無聊、剛好→心流」原則
4. **漸進式揭露**：環境細節由淺入深，先建立氛圍再揭示挑戰

## 品質檢核

- [ ] 環境氛圍描述是否具體且有畫面感？
- [ ] 三個危險要素是否與主題高度相關？
- [ ] 危險要素是否具備足夠細節供 Boss 設計參考？
- [ ] 玩家目標是否清晰且與環境互動？
- [ ] 輸出是否為繁體中文？

## 使用範例

### 範例指令
```
請使用 @level-designer 為主題「深海科研站」設計關卡環境
```

### 範例輸出片段
```markdown
# 關卡設計：深海科研站

## 環境氛圍
深海 3000 公尺的廢棄科研站，金屬走廊被生物發光藻類覆蓋，
發出幽暗的藍綠色光芒。水壓警報間歇響起，暗示結構正在崩潰...

## 環境危險要素
### 危險 1：水壓裂縫
走廊牆壁出現高壓水柱，玩家必須在限時內找到密封閥門...
```
