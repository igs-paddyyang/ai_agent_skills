# skill-creator（技能建立器）

**內建最佳實踐的 CLI 技能自動化建立工具。**

## 功能說明

skill-creator 自動化完成建立新 CLI 技能的整個工作流程。它引導您完成腦力激盪、套用標準化範本、驗證內容品質，並處理安裝——全程遵循 Anthropic 官方最佳實踐。

## 主要特色

- 🎯 **互動式腦力激盪** — 協作式會議，定義技能用途與範圍
- ✨ **範本自動化** — 零手動配置的自動檔案產生
- 🔍 **品質驗證** — 內建 YAML、內容品質與寫作風格檢查
- 📦 **彈性安裝** — 可選擇僅儲存庫、全域或混合安裝
- 📊 **視覺化進度條** — 即時進度指示器（例如 `[████████████░░░░░░] 60% - 步驟 3/5`）
- 🔗 **提詞工程師整合** — 可選使用 prompt-engineer 技能增強

## 何時使用

適用於以下情境：
- 依照官方標準建立新的 CLI 技能
- 以自訂功能擴充 CLI
- 將領域知識封裝為可重用的技能格式
- 將重複性 CLI 任務自動化為自訂技能
- 在本地或全域安裝技能

## 安裝方式

### 前置條件

此技能是 `antigravity-awesome-skills` 儲存庫的一部分。

### 全域安裝（建議）

透過符號連結安裝，使技能在任何地方都可使用：

```bash
# 建立符號連結
ln -sf "$(pwd)/.agent/skills/skill-creator" ~/.agent/skills/skill-creator
```

**全域安裝的好處：**
- 在任何目錄都可使用
- 執行 `git pull` 時自動更新
- 不需要額外配置檔

### 僅儲存庫安裝

若偏好僅在此儲存庫中使用，無需額外安裝。在工作目錄中即可使用此技能。

## 使用方式

### 基本技能建立

直接要求 CLI 建立新技能：

```
「建立一個用於除錯 Python 錯誤的新技能」
「建立一個協助 git 工作流程的技能」
```

技能將以視覺化進度追蹤引導您完成：
1. **腦力激盪**（20%）— 定義用途、觸發條件與類型
2. **提詞增強**（40%，選用）— 使用 prompt-engineer 技能增強
3. **檔案產生**（60%）— 從範本建立檔案
4. **驗證**（80%）— 檢查品質與標準
5. **安裝**（100%）— 選擇本地、全域或兩者

### 進階用法

#### 建立程式碼生成技能

```
「建立一個從描述生成 React 元件的程式碼技能」
```

#### 建立文件技能

```
「打造一個從程式碼撰寫 API 文件的技能」
```

## 檔案結構

建立技能時，會產生以下結構：

```
.agent/skills/your-skill-name/
├── SKILL.md              # 主要技能指令（1.5-2k 字）
├── README.md             # 面向使用者的文件
├── references/           # 詳細指南（每份 2k-5k 字）
├── examples/             # 可運行的程式碼範例
└── scripts/              # 可執行的工具腳本
```

## 驗證項目

每個建立的技能都會自動驗證：
- ✅ **YAML 前置資料** — 必要欄位與格式
- ✅ **描述格式** — 第三人稱、觸發短語
- ✅ **字數限制** — 理想 1,500-2,000，上限 5,000
- ✅ **寫作風格** — 祈使句形式，無第二人稱
- ✅ **漸進式揭露** — 適當的內容組織

## 疑難排解

### 「範本未找到」錯誤
確認您在正確的儲存庫目錄中。

### 「平台未偵測到」警告
選擇「僅儲存庫」安裝，或在設定時手動指定平台。

### 驗證失敗
審閱輸出中的建議，選擇自動修正常見問題，或手動編輯複雜問題。
