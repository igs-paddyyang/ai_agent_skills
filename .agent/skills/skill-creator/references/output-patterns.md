# 輸出模式

當技能需要產出一致且高品質的輸出時，使用以下模式。

## 範本模式

為輸出格式提供範本。根據需求調整嚴格程度。

**嚴格要求（如 API 回應或資料格式）：**

```markdown
## 報告結構

務必使用此精確的範本結構：

# [分析標題]

## 執行摘要
[一段式關鍵發現概述]

## 關鍵發現
- 發現 1 及支持數據
- 發現 2 及支持數據
- 發現 3 及支持數據

## 建議事項
1. 具體可執行的建議
2. 具體可執行的建議
```

**彈性指引（當需要適應性調整時）：**

```markdown
## 報告結構

以下是合理的預設格式，但請依據判斷調整：

# [分析標題]

## 執行摘要
[概述]

## 關鍵發現
[根據實際發現調整章節]

## 建議事項
[針對具體情境量身定制]

依據具體分析類型，視需要調整章節。
```

## 範例模式

當輸出品質取決於參考範例時，提供輸入/輸出配對：

```markdown
## 提交訊息格式

依照以下範例產生提交訊息：

**範例 1：**
輸入：新增使用 JWT 令牌的使用者驗證
輸出：
feat(auth): 實作基於 JWT 的驗證機制

新增登入端點與令牌驗證中介軟體

**範例 2：**
輸入：修復報表中日期顯示不正確的錯誤
輸出：
fix(reports): 修正時區轉換中的日期格式

在報表產生過程中統一使用 UTC 時間戳記
```

範例比單純的描述更能幫助 AI 理解期望的風格與細節程度。
