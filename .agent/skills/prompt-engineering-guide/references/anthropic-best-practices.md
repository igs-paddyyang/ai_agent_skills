# Anthropic Prompt 最佳實踐摘要

本文件整理 Anthropic 官方推薦的 prompt 設計原則，適用於 Claude 系列模型。

## 核心原則

### 1. 清楚明確（Be Clear and Direct）

AI 無法讀心。直接說出你要什麼，不要暗示。

```
# ❌ 暗示
看看這段程式碼，有什麼想法？

# ✅ 明確
檢查這段程式碼的以下面向：
1. 潛在的 Bug
2. 效能問題
3. 安全漏洞
列出每個問題並提供修正建議。
```

### 2. 提供範例（Use Examples）

範例比規則更有效。給 AI 看你期望的輸入和輸出。

```
將以下 git commit 訊息轉換為 changelog 條目：

範例：
- 輸入：`feat: add user authentication`
- 輸出：`✨ 新增使用者認證功能`

- 輸入：`fix: resolve memory leak in cache`
- 輸出：`🐛 修復快取記憶體洩漏問題`

現在轉換：
- 輸入：`refactor: simplify database connection logic`
```

### 3. 給 AI 角色（Assign a Role）

角色設定幫助 AI 調整回應的專業度和風格。

```
你是一位資深 Python 開發者，專精於 Clean Architecture 和測試驅動開發。
請用繁體中文回答，程式碼註解也用繁體中文。
```

### 4. 結構化輸出（Structure Your Output）

明確指定輸出格式，減少不確定性。

```
請用以下格式回答：

## 問題分析
（一句話描述問題）

## 根本原因
（技術層面的原因）

## 修正方案
（具體的程式碼修改）

## 預防措施
（避免再次發生的建議）
```

### 5. 思考鏈（Chain of Thought）

對複雜問題，要求 AI 逐步思考。

```
分析這個架構決策時，請依序考慮：
1. 目前的痛點是什麼？
2. 有哪些替代方案？
3. 每個方案的優缺點？
4. 推薦哪個方案？為什麼？
```

### 6. 限制範圍（Set Boundaries）

告訴 AI 什麼不要做，和什麼要做一樣重要。

```
回答時：
- 只討論 Python 3.9+ 的語法
- 不要建議使用第三方套件（除非明確要求）
- 不要重複問題描述
- 如果不確定，說「我不確定」而非猜測
```

## Prompt 結構模板

### 通用模板

```
[角色設定]
[任務描述]
[輸入資料]
[輸出格式]
[限制條件]
[範例（選用）]
```

### 程式碼審查模板

```
請審查以下程式碼：

```python
{code}
```

檢查項目：
1. 正確性：邏輯是否正確？
2. 效能：有無效能瓶頸？
3. 安全性：有無安全漏洞？
4. 可讀性：命名和結構是否清楚？
5. 測試性：是否容易測試？

對每個問題，提供：
- 問題位置（行號）
- 問題描述
- 修正建議（含程式碼）
```

### 重構建議模板

```
我想重構以下模組：

目前的問題：
- {問題 1}
- {問題 2}

目標：
- {目標 1}
- {目標 2}

限制：
- 不能改變公開 API
- 必須向後相容

請提供重構計畫，包含步驟和每步的程式碼變更。
```

## 常見錯誤

| 錯誤 | 為什麼不好 | 修正 |
|------|-----------|------|
| 「幫我寫程式」 | 太模糊，AI 不知道寫什麼 | 「用 Python 寫一個函式，接受 email 字串，回傳是否有效」 |
| 「寫得好一點」 | 「好」沒有定義 | 「改善命名、加上型別標註、加上 docstring」 |
| 「不要有 Bug」 | 不可能的要求 | 「加上邊界值檢查和錯誤處理」 |
| 一次問太多問題 | AI 容易遺漏 | 分成多個獨立的 prompt |
