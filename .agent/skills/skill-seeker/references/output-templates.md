# 產出範本與範例

> Phase 3（技能草稿產生）時載入此文件，作為各檔案的產出範本。

---

## 1. SKILL.md 範本

```yaml
---
name: <skill-name>
description: >
  <功能描述>。<觸發情境描述>。
  當使用者提到 <關鍵字1>、<關鍵字2>、<關鍵字3> 時，請務必使用此技能。
---

# <技能標題>

<1-2 句核心說明：這個技能提供什麼知識或能力>

## 使用時機

- <觸發情境 1>
- <觸發情境 2>
- <觸發情境 3>

## 核心知識

<從文件中精簡提取的關鍵知識，聚焦於 AI 如何運用>

### <主題 1>

<精簡的核心內容>

### <主題 2>

<精簡的核心內容>

## 附帶資源

| 檔案 | 用途 | 何時載入 |
|------|------|---------|
| references/<file1>.md | <用途說明> | <載入時機> |
| references/<file2>.md | <用途說明> | <載入時機> |
```

### SKILL.md 撰寫要點

- description 公式：`[做什麼] + [什麼時候用 / 觸發關鍵字]`
- 核心知識區塊：用表格和列表提升可掃描性，避免長段落
- 附帶資源索引：每個 references/ 檔案都要列出，說明何時載入
- 語氣：祈使句，解釋為什麼重要

## 3. references/ 檔案範本

每個 references/ 檔案的開頭格式：

```markdown
# <檔案標題>

> <一句話說明此檔案的涵蓋範圍>
> 來源：<原始來源 URL 或檔案路徑>

---

<正文內容>
```

### 內容組織原則

- 保留原始文件的結構層次（標題、子標題）
- 程式碼區塊保持完整，不截斷
- 表格優先於長段落
- 每個檔案聚焦單一主題，不超過 300 行
- 超過 300 行時，進一步拆分為子主題檔案

## 4. 產出範例：FastAPI 技能

假設從 `https://fastapi.tiangolo.com/` 轉換，預期產出：

```
fastapi/
├── SKILL.md                    # ~150 行：FastAPI 核心概念 + 路由 + 依賴注入
├── README.md                   # 版本 0.1.0，來源標註
└── references/
    ├── api-reference.md        # 路由裝飾器、請求/回應模型、狀態碼
    ├── usage-guide.md          # 建立第一個 API、路徑參數、查詢參數
    ├── setup-guide.md          # 安裝、專案結構、uvicorn 啟動
    └── examples.md             # CRUD 範例、認證範例、WebSocket 範例
```

SKILL.md 的 description 範例：
```
FastAPI 框架的完整知識庫，涵蓋路由設計、依賴注入、請求驗證與非同步處理。
當使用者提到 FastAPI、建立 API、REST API、Python Web 框架、路由設計時，請務必使用此技能。
```
