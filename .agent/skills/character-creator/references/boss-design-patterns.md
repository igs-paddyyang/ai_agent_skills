# Boss 設計模式參考

## Boss 戰鬥階段設計

### 三階段模式（經典）
1. **第一階段（100%-60% HP）**：基礎攻擊模式，讓玩家學習規律
2. **第二階段（60%-30% HP）**：新增技能或強化現有技能
3. **第三階段（30%-0% HP）**：狂暴模式，節奏加快

### 環境互動模式
- Boss 利用環境要素作為攻擊手段
- 玩家可利用環境要素反擊 Boss
- 隨階段推進，環境互動方式改變

## 邏輯一致性檢查表

### 環境-技能對應矩陣

| 環境類型 | 合理技能 | 不合理技能 |
|---------|---------|-----------|
| 水下 | 水壓操控、電擊、墨汁遮蔽 | 火焰、沙暴 |
| 火山 | 岩漿噴射、地震、高溫灼燒 | 冰凍、水攻 |
| 太空 | 重力操控、真空吸引、雷射 | 毒霧、洪水 |
| 叢林 | 藤蔓束縛、毒素、召喚野獸 | 機械攻擊 |
| 冰原 | 冰凍、暴風雪、滑冰衝撞 | 火焰（除非有特殊敘事） |

### 敘事動機模板

1. **守護者型**：守護某個秘密或寶物
2. **入侵者型**：外來勢力佔據此地
3. **變異者型**：原本的居民因環境變異而異化
4. **創造者型**：此環境的創造者或管理者
5. **囚禁者型**：被封印在此地的古老存在

## 能力設計原則

### 可讀性
- 每個技能應有明確的視覺預告動作
- 玩家應能在 2-3 次嘗試後學會規避

### 公平性
- 避免無法規避的攻擊
- 每個技能都應有對應的反制方式

### 記憶點
- 至少一個標誌性技能讓玩家印象深刻
- 技能命名應與主題和角色背景呼應
