# 文件摘要輸出模板

產出摘要時，依照以下結構組織內容。根據文件類型彈性增減區塊，但核心骨架保持一致。

此模板與 `websearch-summarizer` 的模板共用相同骨架，差異在於元資料欄位（本地檔案路徑 vs URL）。

---

## 模板結構

```markdown
# [文件名稱] — 文件摘要

**來源檔案**: [檔案路徑]
**文件類型**: [Markdown / PDF / 程式碼 / 純文字 / ...]
**原文長度**: [X 字 / X 行]
**整理日期**: [YYYY-MM-DD]

---

## 一句話摘要

[用一句話概括這份文件的核心內容]

---

## 核心概念

[2-3 句解釋文件的核心主題、解決的問題或傳達的訊息]

---

## 內容總覽

| 章節/模組 | 說明 | 頁數/行數 |
|-----------|------|----------|
| ... | ... | ... |

---

## [關鍵細節區塊]

[根據文件性質展開重要區塊]

---

## 行動建議

1. [具體可執行的下一步 1]
2. [具體可執行的下一步 2]
3. [具體可執行的下一步 3]
```

---

## 各類文件的區塊組合

### 規格文件（spec）

1. 標題與元資料
2. 一句話摘要
3. 核心概念
4. 需求總覽（表格）
5. 架構設計摘要
6. 實作步驟（編號清單）
7. 測試計畫摘要
8. 已知風險 / 開放問題
9. 行動建議

### 設計文件

1. 標題與元資料
2. 一句話摘要
3. 核心概念
4. 問題定義
5. 方案比較（表格）
6. 最終決策與理由
7. 行動建議

### API 文件

1. 標題與元資料
2. 一句話摘要
3. 核心概念
4. 端點總覽（表格：方法、路徑、說明）
5. 認證方式
6. 關鍵端點詳情（請求/回應範例）
7. 錯誤碼（表格）
8. 行動建議

### 程式碼檔案

1. 標題與元資料（含語言、行數）
2. 一句話摘要
3. 核心概念
4. 模組/類別結構（表格）
5. 核心函式（表格：名稱、參數、用途）
6. 依賴關係
7. 行動建議

### 會議紀錄 / 筆記

1. 標題與元資料（含日期、參與者）
2. 一句話摘要
3. 決議事項（編號清單）
4. 行動項目（表格：負責人、事項、期限）
5. 待討論 / 開放問題
6. 行動建議

### 通用長文

1. 標題與元資料
2. 一句話摘要
3. 核心概念
4. 內容總覽（表格）
5. 關鍵論點 / 數據
6. 結論
7. 行動建議

---

## 格式偏好

- 表格 > 長段落（提升可掃描性）
- 具體數據 > 模糊描述
- 原文術語保留英文，說明用繁體中文
- 每個區塊都要有實質內容，寧可省略也不要空殼
- 摘要長度控制在原文的 10-20%
