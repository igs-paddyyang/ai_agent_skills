# 🚀 AI Agent 實戰系列：GDD 自動化生成實作計畫 (v1.0)

本計畫旨在透過 `antigravity-awesome-skills` 技能庫，將 AI 轉化為具備「遊戲研發專業」的代理人，實現自動化產出高品質遊戲設計文件 (GDD) 的目標。

---

## 📋 1. 執行目標
*   **自動化掛載**：動態調用專業遊戲研發技能。
*   **鏈式處理**：實現場景設計與角色設計的邏輯關聯。
*   **規格化產出**：生成符合工業標準的 Markdown GDD 文件。

---

## 🏗️ 2. 四階段執行計畫

### 第一階段：環境佈建與基礎設施
*   **核心任務**：建立隔離開發環境與資源同步。
*   **具體步驟**：
    1.  **環境初始化**：建立 Python 虛擬環境並安裝 `google-genai`, `python-dotenv` 套件。
    2.  **相依性安裝**：確保使用的 SDK 版本支援最新 `gemini-2.0-flash` 模型。
    3.  **金鑰配置**：設定 `.env` 檔案中的 `GOOGLE_API_KEY`。
    4.  **技能庫同步**：獲取最新的技能體系。

### 第二階段：技能掛載與連通性驗證
*   **核心任務**：驗證系統對於外部技能定義的偵測與讀取能力。
*   **具體步驟**：
    1.  **建立載入器**：實作 `loader.py` 用於掃描技能標籤。
    2.  **路徑檢測 (TEST-01)**：確保能正確識別並讀取技能檔案。
    3.  **API 校準 (TEST-02)**：執行 `list_models.py` 檢查可用模型清單，並使用 `test_api.py` 針對 `gemini-2.0-flash` 進行預熱測試。

### 第三階段：GDD 生成器開發與產出轉向 (Development)
*   **核心任務**：運用「思維鏈」架構串接多種專業技能。
*   **具體步驟**：
    1.  **生成邏輯開發**：實作 `gdd_generator.py` 處理多步驟呼叫。
    2.  **自動化編排**：串接關卡設計與角色編輯技能。
    3.  **產出轉向**：將 GDD 檔案輸出路徑由 `docs/` 修改為 `reports/` 並加上日期戳記。
    4.  **主題化產出**：檔案命名為 `reports/GDD_{主題}_{日期}.md`。

### 第四階段：目錄標準化重構 (Refactoring)
*   **核心任務**：確保專案符合全域規範 (Rule 4)，完成物理遷移。
*   **具體步驟**：
    1.  **文件遷移**：將系統設計文件移入 `docs/` 並加上版號。
    2.  **腳本分類**：將測試類腳本從 `scripts/` 移入 `tests/`。
    3.  **路徑校準**：更新 `gdd_generator.py` 與 `qa_validator.py` 中的內部引用路徑。
    4.  **終端驗證**：確保重構後 `py agent_skills/src/gdd_generator.py` 依然能正確運行。
