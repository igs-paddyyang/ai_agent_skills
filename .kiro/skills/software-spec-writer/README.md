# 軟體工程規格文件撰寫師（Software Spec Writer）

> 將專案構想轉化為結構完整、可執行的規格文件，涵蓋規格設計與執行計畫。

## 功能說明

扮演資深軟體架構師，產出「一份文件，兩個視角」的規格文件。Part I（§1-5）為規格設計：需求分析 → 系統設計 → 實作路徑 → 測試指南 → 問題追蹤。Part II（§6-8）為執行計畫：工項分解 → 時間估算 → Checklist → 風險管理。避免 spec 和 plan 分成兩份文件導致不同步。

## 使用方式

觸發此技能的方式：

```
「幫我寫一份 ArkBot Skill Factory 的規格文件」
「產出這個功能的 spec」
「撰寫系統設計文件」
「幫我做實作指南和任務分解」
```

## 產出結構

```markdown
# 🚀 [專案名稱] (版本號)

## ═══ Part I：規格設計 ═══
## 📋 1. 需求分析
## 🏗️ 2. 系統設計
## 🛠️ 3. 實作路徑
## ✅ 4. 驗證與測試指南
## 🐛 5. 已知問題與修正紀錄

## ═══ Part II：執行計畫 ═══
## 📊 6. 任務分解與時間估算
## ☑️ 7. 執行 Checklist
## ⚠️ 8. 風險管理與應對
```
