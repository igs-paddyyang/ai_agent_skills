# 摘要文件輸出模板

產出摘要時，依照以下結構組織內容。根據來源性質彈性增減區塊，但核心骨架保持一致。

---

## 模板結構

```markdown
# [資源名稱] — 文件摘要

**來源**: [URL](URL)
**版本**: [版本號或日期，若有]
**授權**: [授權類型，若有]
**整理日期**: [YYYY-MM-DD]

---

## 一句話摘要

[用一句話概括這個資源的核心價值]

---

## 核心概念

[2-3 句解釋這個資源的核心理念、運作方式或解決的問題]

---

## [分類/功能/模組] 總覽

| 分類 | 說明 | 代表項目 |
|------|------|---------|
| ... | ... | ... |

---

## [關鍵細節區塊 1]

[根據內容性質展開，例如：安裝方式、API 說明、架構設計、使用方式等]

## [關鍵細節區塊 2]

[可以有多個關鍵細節區塊]

---

## 與本專案的關係

[選用。如果跟 Agent Skills Factory 有關聯，說明如何應用]

---

## 行動建議

1. [具體可執行的下一步 1]
2. [具體可執行的下一步 2]
3. [具體可執行的下一步 3]
```

## 區塊使用指南

### 必要區塊

| 區塊 | 說明 |
|------|------|
| 標題與元資料 | 永遠包含。來源 URL 是最重要的追溯資訊 |
| 一句話摘要 | 永遠包含。讓讀者 3 秒內決定是否繼續閱讀 |
| 核心概念 | 永遠包含。比一句話摘要更深一層的理解 |
| 行動建議 | 永遠包含。摘要的價值在於驅動行動 |

### 選用區塊

| 區塊 | 何時使用 |
|------|---------|
| 分類/功能總覽 | 來源有明確的分類體系或多個功能模組時 |
| 安裝方式 | 來源是工具、套件或框架時 |
| API / 介面 | 來源提供 API 或程式介面時 |
| 架構設計 | 來源是系統或專案，有明確架構時 |
| 相容性 | 來源支援多平台或多工具時 |
| 安全機制 | 來源有安全相關的設計時 |
| 與本專案的關係 | 來源跟 Agent Skills Factory 有直接關聯時 |

### 格式偏好

- 表格 > 長段落（提升可掃描性）
- 具體數據 > 模糊描述（「1,272 個技能」而非「大量技能」）
- 原文術語保留英文，說明用繁體中文
- 每個區塊都要有實質內容，寧可省略也不要空殼

## 範例：GitHub Repo 摘要

適用於 GitHub 開源專案的典型區塊組合：

1. 標題與元資料（含 star 數、授權）
2. 一句話摘要
3. 核心概念
4. 功能/模組總覽（表格）
5. 安裝方式（程式碼區塊）
6. 專案結構（表格）
7. 與本專案的關係（選用）
8. 行動建議

## 範例：技術文件摘要

適用於 API 文件、技術規格的典型區塊組合：

1. 標題與元資料
2. 一句話摘要
3. 核心概念
4. API / 介面總覽（表格）
5. 關鍵端點或功能（程式碼範例）
6. 限制與注意事項
7. 行動建議

## 範例：產品/服務摘要

適用於產品頁面、SaaS 服務的典型區塊組合：

1. 標題與元資料（含定價資訊）
2. 一句話摘要
3. 核心概念
4. 功能分類（表格）
5. 定價方案（表格）
6. 相容性 / 整合
7. 行動建議
