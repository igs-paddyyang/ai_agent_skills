# GDD 模板結構

產出遊戲企劃文件時，依照以下模板結構組織內容。根據遊戲類型彈性調整各章節深度，但整體骨架保持一致。

---

## 章節必填/選填指南

| 章節 | full 模式 | onepager 模式 | 說明 |
|------|----------|--------------|------|
| 遊戲概述 | 必填 | 必填（濃縮） | 所有遊戲都需要 |
| 核心玩法 | 必填 | 必填（濃縮） | GDD 的核心 |
| 系統機制 | 必填 | 省略 | 需要數值表格 |
| 關卡與內容 | 必填 | 省略 | 無關卡概念的遊戲可簡化 |
| 敘事與世界觀 | 必填 | 省略 | 無劇情的遊戲可簡化 |
| 美術風格 | 必填 | 必填（濃縮） | 文字描述即可 |
| 音效設計 | 必填 | 省略 | 可簡化 |
| 技術規格 | 必填 | 省略 | 視複雜度調整 |
| 營運與商業模式 | 必填 | 必填（濃縮） | 商業可行性 |
| 開發計畫 | 必填 | 省略 | 視需求調整 |
