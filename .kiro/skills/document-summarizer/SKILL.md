# 文件摘要師（Document Summarizer）

讀取本地長文件，產出結構化的 Markdown 摘要。適合快速理解長篇規格文件、技術文件、設計文件或程式碼檔案的全貌。

## 使用時機

- 使用者提供一份或多份本地文件，想快速掌握重點
- 長篇規格文件需要濃縮為可行動的摘要
- 需要為團隊成員整理文件的核心內容

## 與 websearch-summarizer 的關係

兩個技能互補，共用相同的輸出結構模板：

| 技能 | 輸入來源 | 適用場景 |
|------|---------|---------|
| websearch-summarizer | URL / 搜尋關鍵字 | 網頁、GitHub Repo、線上文件 |
| document-summarizer | 本地檔案 | Markdown、PDF、程式碼、文字檔 |

## 工作流程

### 步驟 1：確認輸入文件

使用者可能提供：

| 輸入方式 | 處理方式 |
|---------|---------|
| 指定檔案路徑 | 直接讀取該檔案 |
| 指定目錄 | 掃描目錄下的文件，列出清單讓使用者選擇 |
| 拖入檔案（PDF、DOCX 等） | 從附件內容讀取 |
| 「整理這個檔案」（指當前開啟的檔案） | 讀取 active editor 的檔案 |

支援的文件類型：
- Markdown（.md）— 直接讀取
- 純文字（.txt、.log）— 直接讀取
- 程式碼（.py、.js、.ts 等）— 讀取並分析結構
- PDF / DOCX — 從附件內容讀取（需使用者拖入）
- JSON / YAML — 讀取並解析結構

若文件過長（>10,000 字），分段讀取並整合。

### 步驟 2：分析文件結構

根據文件類型採用不同的分析策略：

| 文件類型 | 分析策略 |
|---------|---------|
| 規格文件（spec） | 提取需求、架構、實作步驟、測試計畫 |
| 設計文件 | 提取問題定義、方案設計、取捨分析、決策 |
| API 文件 | 提取端點、參數、回應格式、錯誤碼 |
| 程式碼檔案 | 提取模組結構、核心函式、依賴關係 |
| 會議紀錄 / 筆記 | 提取決議事項、行動項目、待辦 |
| 通用長文 | 提取主題、論點、結論、關鍵數據 |

### 步驟 3：產出結構化摘要

依照摘要模板結構產出 Markdown 文件。核心區塊：

1. **標題與元資料** — 來源檔案路徑、文件類型、字數/行數、整理日期
2. **一句話摘要** — 用一句話概括文件的核心內容
3. **核心概念** — 2-3 句解釋文件的核心主題或解決的問題
4. **內容總覽** — 用表格整理文件的主要章節或模組
5. **關鍵細節** — 依文件性質展開重要區塊
6. **行動建議** — 讀完這份摘要後，具體的下一步行動

詳細的模板結構與各類文件的區塊組合，參考 `references/output-template.md`。

#### 寫作原則

- 用繁體中文撰寫，技術名詞保留英文
- 表格優先於長段落，提升可掃描性
- 保留原始文件的關鍵數據（版本號、數量、日期等）
- 不逐段翻譯或複製，而是重新組織資訊結構
- 行動建議要具體可執行
- 摘要長度控制在原文的 10-20%

### 步驟 4：儲存與回報

1. 若使用者指定輸出路徑 → 存到指定位置
2. 若未指定 → 預設存到 `docs/<原檔名>-summary.md`
3. 向使用者回報：
   - 檔案路徑
   - 一句話摘要
   - 原文長度 vs 摘要長度
   - 若有無法處理的部分，說明原因

## 多文件模式

當使用者提供多份文件時：

1. **獨立摘要** — 每份文件各產出一份摘要（預設）
2. **整合摘要** — 將多份文件整合為一份綜合摘要（使用者要求時）

整合摘要額外包含：
- 文件間的關聯性分析
- 共同主題與差異點
- 整合後的行動建議

## 邊界案例

- **空白或極短文件**（<100 字）→ 直接顯示內容，不需要摘要
- **二進位檔案** → 說明無法處理，建議轉換為文字格式
- **加密或受保護的 PDF** → 說明限制，建議使用者提供解鎖版本
- **超大文件**（>50,000 字）→ 分段處理，優先摘要前 80% 核心內容
