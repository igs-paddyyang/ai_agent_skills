# 🧪 ClawdBot 對話測試指南 (Telegram)

本指南旨在幫助開發者與使用者驗證 **ClawdBot** 的核心功能，確保系統在不同情境下皆能穩定運行。

---

## 🚀 1. 快速開始測試
在開始對話前，請確保：
1. `bot_main.py` 已正確執行。
2. 您已在 Telegram 中與您的機器人啟動對話（點擊 `/start`）。

---

## 🧪 2. 核心測試案例 (Core Test Cases)

### 案例 2.1：初始化驗證
*   **使用者輸入**：`/start`
*   **預期行為**：
    1. 機器人立即回傳歡迎訊息。
    2. 檢查文字中是否包含 `(v1.0)` 且格式正確（無 Markdown 解析報錯）。

### 案例 2.2：主動網頁獵取 (Research Intent)
*   **使用者輸入**：`幫我分析這篇網址：https://www.google.com`
*   **預期行為**：
    1. 顯示 `🧠 正在思考...`
    2. 更新為 `🔍 偵測到網址，正在獵取中...`
    3. 更新為 `🧠 內容已獲取，正在進行智慧摘要與智庫封存...`
    4. 最後輸出結構化的繁體中文摘要，並附帶原始來源連結。

### 案例 2.3：資料庫快取驗證 (Cache First)
*   **操作步驟**：對同一個網址（如案例 2.2 的網址）再次發送分析請求。
*   **預期行為**：
    1. 摘要內容應包含 `(來自快取)` 字樣（若有實作標註）。
    2. 生成速度應顯著快於第一次（無需執行網路抓取）。

### 案例 2.4：一般閒聊模式 (Casual Intent)
*   **使用者輸入**：`你好，你覺得今天台北天氣如何？`
*   **預期行為**：
    1. 機器人辨識為非研究意圖。
    2. 顯示 `💬 正在生成回應...`
    3. 給予一個簡潔、友善的對話回覆。

---

## 🛡️ 3. 邊界與異常測試 (Robustness)

### 案例 3.1：無效網址測試
*   **使用者輸入**：`https://this-is-not-a-real-site-123.com`
*   **預期行為**：機器人應回報獵取失敗，而非當機或無限等待。

### 案例 3.2：特殊字元穩定性測試
*   **使用者輸入**：發送一段包含大量 `_ * [ ] ( ) ~ # + - = | { } . !` 的文字。
*   **預期行為**：Bot 應能正確處理（透過 `escape_markdown_v2`）並回應，不應出現 `Can't parse entities` 錯誤。

---

## 📂 4. 驗證資料庫存儲
測試完成後，您可以檢查根目錄下的 `clawdbot/brain.db`：
*   **`raw_crawls`**：應該有您剛才抓取的完整網頁內容。
*   **`memories`**：應該存有 AI 生成的精華摘要。
